Why is Branching Necessary

Branching in Git allows developers to work independently on different features, bug fixes, or experiments without interfering with the main codebase. It ensures that the *main branch* (often called `main` or `master`) stays stable and free of incomplete or potentially buggy code. Teams can collaborate more efficiently by using branches for specific tasks and then merging changes when ready.

Common Branching Scenarios

  • Developing new features without affecting the main project
  • Fixing bugs while the main codebase remains unchanged
  • Testing out experimental code without risking breaking the stable version

Creating a New Branch

git checkout -b <branch-name>

This command creates a new branch and switches to it.

Switching Between Branches

git checkout <branch-name>

This command allows you to switch to an existing branch.

Listing All Branches

git branch

This command shows all branches in your repository.

Merging Branches

git checkout main
git merge <branch-name>

Merge changes from another branch into the main branch.

Deleting a Branch

git branch -d <branch-name>

After merging, delete the branch locally using this command.

Pushing Branches to Remote

git push origin <branch-name>

To share your branch with others, push it to the remote repository.
