Making Changes in a Project

git status

The git status command shows the current status of your Git repository. It lets you see which files have been modified, which are unstaged, and which are staged for commit.

git add <filename>

This command is used to add specific files to the staging area. For example, git add index.html will add just that file for commit.

git add .

This command stages all changed files in the project for commit. The difference between git add <filename> and git add . is that the latter stages all changes, while the former only stages a single file.

git commit -m "description of changes"

git push

This command is used to send your committed changes from your local repository to a remote repository, like GitHub. Example: git push origin main.
